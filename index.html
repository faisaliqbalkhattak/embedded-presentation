What is CaDiCaL?

CaDiCaL (pronounced “radical”) is an open-source, state-of-the-art CDCL-based SAT solver developed by Armin Biere and his team at the University of Freiburg (formerly JKU Linz).

Design Philosophy: Build a CDCL solver that is easy to understand and modify, while remaining competitive with the fastest solvers in the world.

It is the successor to the Lingeling solver family and has been published at CAV 2024 as a tool paper (LNCS vol. 14681).

CaDiCaL at a Glance

  • Language: C++ (clean, well-documented)
  • License: MIT
  • Current Version: 3.0.0
  • Lines of Code: ~40,000+
  • Interface: IPASIR-2 compliant API
Key Achievement: Winner of multiple SAT Competition gold medals across main and incremental tracks.

What is SAT?

Boolean Satisfiability (SAT): Given a propositional formula in Conjunctive Normal Form (CNF), determine if there exists a truth assignment that makes the entire formula true.

CNF Example

c Example: 2 variables, 2 clauses
p cnf 2 2
 1  2  0       ← (x₁ ∨ x₂)
-1     0       ← (¬x₁)

Solution: x₁ = false, x₂ = true  ✓

Why SAT Matters

  • Hardware Verification: Checking circuit correctness
  • Software Testing: Finding bugs via model checking
  • AI Planning: Encoding planning problems as SAT
  • Cryptanalysis: Breaking simple ciphers
  • Scheduling: Constraint satisfaction in timetabling

SAT is the first proven NP-complete problem (Cook-Levin theorem, 1971). Despite worst-case exponential complexity, modern solvers handle instances with millions of variables.

The DPLL Algorithm (1962)

Davis-Putnam-Logemann-Loveland — the foundation of SAT solving.

  • Unit Propagation: If a clause has one unset literal, force it true.
  • Pure Literal Elimination: If a variable appears with only one polarity, assign it.
  • Splitting / Branching: Pick an unassigned variable, try both values.
  • Chronological Backtracking: On conflict, undo the last decision.

Limitation: Repeats the same mistakes — no memory of past conflicts.

CDCL: The Modern Approach (1996+)

Conflict-Driven Clause Learning (Marques-Silva & Sakallah).

  • Conflict Analysis: On conflict, analyze why it happened.
  • Clause Learning: Add a new “learned clause” to prevent repetition.
  • Non-Chronological Backjumping: Jump back to the actual cause, skipping irrelevant decisions.
  • Activity-Based Heuristics: Focus on variables involved in recent conflicts (VSIDS / EVSIDS).
Result: Orders of magnitude faster than DPLL on structured instances.

Core CDCL Loop

The main loop that powers CaDiCaL and all modern SAT solvers.

1
Decide: Pick an unassigned variable using EVSIDS scoring.
2
Propagate: Apply BCP via two-watched-literal scheme.
3
Conflict? If a clause becomes empty, a conflict is detected.
4
Analyze: Trace the implication graph to find the 1-UIP cut.
5
Learn & Backjump: Add the learned clause, backtrack to assertion level.
6
Restart: Periodically restart to escape unproductive regions.

Key Data Structures

  • Two-Watched Literals: Efficient unit propagation — only inspect a clause when a watched literal becomes false.
  • Implication Graph: Tracks which assignments were forced by which clauses. Enables conflict analysis.
  • EVSIDS Scores: Exponential decay activity scores. Variables in recent conflicts get higher priority.
  • Trail: Stack of assignments with decision levels. Supports backjumping.
  • Clause Database: Arena-based allocation for cache efficiency.

Search Heuristics

What makes CaDiCaL’s search strategy stand out.

  • EVSIDS (Score-based): Exponential VSIDS with configurable decay factor (default: 0.95).
  • Phase Saving: Remember last assigned value of each variable for faster convergence.
  • Focused & Stable Modes: Alternates between aggressive exploration and exploitation phases.
  • Reluctant Doubling Restarts: Luby-style restart sequence in stable mode.
Configuration: CaDiCaL has 150+ tunable parameters, but ships with carefully optimized defaults.

Clause Management

How CaDiCaL organizes and prunes its learned clause database.

  • Glue / LBD Score: Measures clause quality — lower glue = more useful. Tier 1 (glue ≤ 2), Tier 2 (glue ≤ 6), Tier 3 (rest).
  • Clause Reduction: Periodically delete low-quality learned clauses to keep the database lean.
  • On-the-fly Subsumption (OTFS): Strengthen existing clauses during conflict analysis.
  • Chronological Backtracking: Optionally avoid full backjumps to preserve useful assignments.

Preprocessing Techniques

Simplification applied before the search begins.

✏️

Bounded Variable Elimination

Resolve away variables when the resulting formula is smaller.

🔎

Subsumption & Strengthening

Remove clauses subsumed by shorter ones. Remove redundant literals.

💡

Failed Literal Probing

Tentatively assign a literal; if it conflicts, the opposite must be true.

Inprocessing Techniques

Simplification interleaved during the search — CaDiCaL’s key advantage.

🔌

Congruence Closure

Detect AND, XOR, ITE gate structures. Merge equivalent variables.

🛠️

SAT Sweeping

Detect equivalent variables via simulation + SAT sub-calls.

🔄

Bounded Variable Addition

Introduce new variables to reduce total literal occurrences.

Key Insight: After a restart, CaDiCaL may run a round of BVE/subsumption before resuming CDCL search.

C++ API Example

CaDiCaL provides a clean, IPASIR-compliant C++ API.

#include "cadical.hpp"

int main() {
  CaDiCaL::Solver solver;

  // Add clause (x₁ ∨ x₂)
  solver.add(1);
  solver.add(2);
  solver.add(0);  // terminate clause

  // Add clause (¬x₁)
  solver.add(-1);
  solver.add(0);

  int res = solver.solve();
  // res == 10 → SATISFIABLE

  int v1 = solver.val(1); // -1 (false)
  int v2 = solver.val(2); //  2 (true)

  return 0;
}

Command-Line Usage

Build and run CaDiCaL from the terminal.

# Build CaDiCaL
$ ./configure && make

# Solve a DIMACS CNF file
$ ./build/cadical example.cnf
s SATISFIABLE
v 1 -2 0

# Generate DRAT proof
$ ./build/cadical input.cnf proof.drat

# Incremental solving via IPASIR
$ solver.assume(3);
$ solver.solve();   // under assumption x₃ = true
$ solver.failed(3); // check if x₃ in unsat core
Return Codes: 10 = SATISFIABLE, 20 = UNSATISFIABLE, 0 = UNKNOWN.

What We Learned

  • CDCL’s Power: Clause learning transforms exponential backtracking into practical solving for structured problems.
  • Engineering Matters: Cache-efficient data structures (arena allocation, watched literals) are crucial for performance.
  • Inprocessing is Key: Interleaving simplification with search yields significant speedups over pure CDCL.
  • Clean Code Wins: CaDiCaL proves that readability and performance are not mutually exclusive.
Takeaway: CaDiCaL exemplifies how decades of SAT research converge into a solver that balances theoretical soundness, engineering excellence, and practical impact.

Future Directions

  • Parallelism: CaDiCaL is single-threaded — portfolio/clause-sharing parallelism could help.
  • Machine Learning: ML-guided branching heuristics could adapt to specific problem families.
  • Parity / Cardinality: Native support for XOR/cardinality constraints (partially present but limited).
  • Hybrid Approaches: Combining CDCL with local search (SLS) for satisfiable instances.
